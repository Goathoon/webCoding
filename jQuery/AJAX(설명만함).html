AJAX 란, 서버에 접근하여 서버가 제공하는 새로운 데이터를 받아와 웹페이지에 표현합니다. 이 기능은 서버스크립트 언어와 함께 사용해야 됨. $.ajax({ //AJAX 시작하기 type: GET 방식인지 POST 방식인지 선택, dataType: json 인지 xml인지 선택, url: 데이터를 전송할 url 입력, data: {보낼 데이터 입력} async: 비동기 방식인지, 동기 방식인지 선택 success: function(data){ //에이잭스 통신 성공시 명령어 입력 } error: function(request, status, error) { //에이잭스 통신 실패시 명령어 입력 } }) --type-- type에서 GET방식은 전송할 정보를 다음과 같이 URL 뒤에 적어 전송한다. eg. www.everdevel.com?id=happycat 위의 url 에서는 GET방식으로 id라는 변수에 happycat이라는 값을 전달한다. 이는 쉽게 노출되므로, 보안에 취약. 또한 길이의 제한도 있어서 데이터가 길어지면 데이터가 전달되지 않는 단점이 있따. POST방식은 네트워크 지식이 조금 필요하다. 컴퓨터가 어떤 규칙으로 통신할 것인지에 대한 프로토콜(규약)이 존재한다. 우리가 사용하는 웹은 http프로토콜을 사용하며, 파일을 주고받기 위해서는 ftp라는 프로토콜을 사용해야 한다. POST 방식에서 이 http프로토콜은 header 와 body로 구분되며, body에 데이터가 전송되고, (사람이 전송되는 정보를 볼 수 없다.???) 또한, POST는 데이터를 입력하는 용도로 사용되며 GET은 데이터를 요청하는 용도로 사용된다. 즉, 이름이라는 데이터를 데이터베이스에 입력할 떄는 POST를 사용하고, 어떠한 게시물의 글들을 불러오기 위해 요청할때는 GET을 사용한다. --dataType-- dataType에는 데이터를 전송하는데 사용할 언어를 선택한다. 이 떄 이 언어는 json 이나 xml이 있다. 이 두 언어의 활용은 서로 다른 프로그래밍 언어 간 통신을 위해 사용되는 언어이다. 즉, 에이잭스에서 사용하는 언어는 제이쿼리(java script)이며, 요청 받는 프로그래밍 언어는 PHP(서버사이드언어)가 된다. jQuery와 PHP는 이와 같이 서로 다른 언어이므로, 서로 통신할 수 있는 언어가 필요하며, 이를 가능하게 하기 위해 json 이나 xml을 사용한다. json은 xml에 비해 속도가 빠르므로 json을 사용하지만, 전송할 데이터가 많은 경우에는 속도가 느리기 떄문에 xml을 사용한다. --url-- url에는 전송한 데이터를 처리할 파일을 기입합니다. 예를들어, 회원가입시 중복체크할 아이디를 전송하고, 그 아이디를 받아 중복 체크할 파일이 idCheck.php라고 하면, url 값에 idCheck.php 파일명과 경로를 적습니다 --data-- data에는 해당 url에 보낼 실제 데이터를 기입합니다. 고객이 입력한 ID가 everdevel이라면 everdevel이라는 값을 기입합니다.또한 everdevel이 ID라는 것을 알 수 있도록 ID라는 변수명도 함께 보냅니다. --async-- 에이잭스는 비동기 통신 방식을 사용합니다. 비동기 통신이란? 데이터를 보내고 그 데이터의 가공된 값 또는 결과물을 받지 않더라도 그 다음 기능을 수행하는 방식입니다. 동기 통신이란? 데이터를 보내고 그 데이터의 가공된 값 또는 결과물을 받아야만 그 다음 기능을 수행하는 방식이다. 비동기 통신은 다음과 같은 상황에 사용됩니다. 자신이 운영하는 웹서비스의 특정 페이지에 접속했을 때 어떤 고객이 몇시에 어떤 웹 브라우저를 사용하여 접속했는지 저장하는 기능을 만들고, 그 기록이 저장된 후 아무런 기능도 실행하지 않도록 할 떄 사용된다. 한편, 동기 통신은 다음과 같은 상황에 사용됩니다. 고객이 [좋아요]버튼을 몇 번 클릭했는지 알려주어야 하는 상황에 사용된다. 클릭하면, 에이잭스를 사용하여 클릭한 값이 전달되고, 전달받은 사이드 언어에서 이를 처리한 후 총 몇건이 클릭되었는지 확인하고 그 값을 전달하며, 그 값을 전달받은 에이잭스는 해당 값을 받아 [좋아요] 옆의 수를 나타내는 엘리먼트에 적용합니다. 에이잭스는 비동기통신방식을 이용하므로, 비동기 통신방식이 기본값으로 지정되어 있으며, 이를 async값에 false를 적용함으로써 동기방식으로 에이잭스를 사용할 수 있습니다. --success: function(data){}-- 에이잭스 통신이 성공했을 때 실행할 명령문을 기입한다. 예를 들어, 중복 체크할 아이디를 전송한 후 그 결과를 받았고, 이미 사용한 아이디라면 다음과 같은 방법을 사용 한다. $('#idCheckStatus').text('이 아이디를 사용할 수 없습니다.'); --error: function(request,status,error){} 에이잭스 통신이 실패했을 떄 실행할 명령문을 기입. 흔히 프로그래머 자신이 어떠한 원인으로 실패했는지 에러코드를 확인하여 수정하기 위해 사용된다. 서버사이드언어를 아직 학습 x 따라서 php(백엔드)에서 배운데용.
